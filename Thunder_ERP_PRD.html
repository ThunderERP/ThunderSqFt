Section 01 · Executive Summary

What we're building

Thunder ERP is a multi-role real estate operations platform covering CRM, banking/loans, HR, finance, and automation. Version 2.0 focuses on two major upgrades: a premium UI overhaul and a complete WhatsApp automation layer powered by Zapier.

Section 05 · Zapier Implementation Guide

Backend API Requirements for Zapier

The ERP backend must expose these webhook endpoints and data APIs for Zapier to consume. All endpoints require Bearer token authentication.

Automation Endpoint Method Response Data
Daily Reminders /api/automation/daily-reminders GET Array of {agent_id, agent_name, wa_number, leads[]}
Follow-Up Alert /webhooks/follow-up-created POST (outgoing webhook) Triggered by ERP on follow-up save
Daily Report /api/reports/daily-summary GET Branch-wise: leads, followups, visits, bookings, revenue
Monthly Report /api/reports/monthly GET ?month=&branch_id= Full monthly stats + PDF URL
Missed Follow-Ups /api/followups/overdue GET Array of overdue follow-ups with agent info
Birthday List /api/contacts/birthdays-today GET Array of {contact_id, name, wa_number, dob}
Anniversary List /api/contacts/anniversaries-today GET Array of {contact_id, name, wa_number, anniversary_date}

🔧 Zapier Zap Architecture

STEP 1 — TRIGGER

Schedule (cron) or Webhook (instant). Zapier catches the event.

STEP 2 — FETCH DATA

Zapier calls Thunder ERP REST API with API key. Gets JSON data.

STEP 3 — FORMAT

Zapier Formatter action builds the WhatsApp message string using template variables.

STEP 4 — SEND

WhatsApp Business app in Zapier sends the message to the recipient number.

Required Zapier Apps

✅ Schedule by Zapier ✅ Webhooks by Zapier ✅ WhatsApp Business ✅ Formatter by Zapier 📌 Zapier Professional plan (for multi-step zaps)

Section 07 · Technical Reference

Database & API Notes

The existing backend is NestJS + Prisma + PostgreSQL. New fields required for automation support.

🗃️ New Database Fields Required

// prisma/schema.prisma — additions for automation

model Contact {
  id   Int @id @default(autoincrement())
  waNumber   String?  // WhatsApp phone number with country code
  dateOfBirth   DateTime? // for birthday automation
  anniversary   DateTime? // for anniversary automation
  waOptIn   Boolean @default(true) // consent flag
}

model AutomationLog {
  id   Int @id @default(autoincrement())
  ruleId   Int
  runAt   DateTime @default(now())
  status   String // "success" | "failed" | "skipped"
  recipientCount Int @default(0)
  errorMessage String?
}

model AutomationRule {
  id   Int @id @default(autoincrement())
  name   String
  isActive   Boolean @default(true)
  schedule   String // cron expression or "webhook"
  messageTemplate String // Handlebars template
  lastRunAt   DateTime?
  lastStatus   String?
}
